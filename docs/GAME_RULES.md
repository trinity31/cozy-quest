## B. 배치 규칙

- 각 부위 = 풍경 내 **1곳만** (중복 X)
- 화면 **5분할 균등 분포**: 좌상·우상·좌하·우하·중앙 → 부위 1개씩
- 부위 간 최소 거리 = 화면 대각선의 **20%**
- 풍경 "숨김 가능 위치 풀": 통·바구니·지붕·벤치·창문·아치·계단·꽃화분·식기·문틈·우산·자전거 등

## C. 가시성 룰

- 부위 면적 = 풍경 픽셀의 **1~3%** (작은 디테일)
- 완전 가림 X (최소 일부 노출)
- 색감 = 풍경 톤과 **약간 다른 색** (시각 단서 = 색 대비)
- **난이도 1~3 등급**:

| 난이도   | 면적 | 색 대비 | 가림 |
| -------- | ---- | ------- | ---- |
| ① 쉬움   | ~3%  | 강함    | 적음 |
| ② 보통   | ~2%  | 보통    | 중간 |
| ③ 어려움 | ~1%  | 약함    | 많음 |

- **목표 평균 발견 시간 = 30~45초** (Sprint1 Risks 기준)

## D. 고양이 정체성

- 풀바디 = 5부위 **색감·무늬 동일** (한 마리)
- 매일 1마리 = 1풍경 = 1가구 보상
- 메타데이터:
  - 이름 (예: 미오, 단팥, 호박)

  - 모티프 (빵집/꽃집/찻집/...)
  - 가구 보상 1개 (3종 택1)

## F. 데이터 스키마 (Supabase, Day 1 참조용)

```sql
-- scenes (풍경 마스터)
scene_id TEXT PRIMARY KEY        -- "market_dawn_001"
title TEXT                       -- "새벽 시장통 광장"
motif TEXT                       -- "market" | "bakery" | ...
difficulty SMALLINT              -- 1 | 2 | 3
image_url TEXT
release_date DATE                -- 어느 날 풀리는 풍경인지

-- cats (고양이 마스터, 1풍경 = 1마리)
cat_id TEXT PRIMARY KEY          -- "cat_001"
scene_id TEXT REFERENCES scenes
name TEXT                        -- "미오"
personality TEXT                 -- "낮잠을 좋아하는"
motif TEXT                       -- 풍경 motif와 매칭
fullbody_image_url TEXT          -- 5부위 다 찾은 후 등장
furniture_options JSONB          -- [{id, name, image_url}, ×3]

-- cat_parts (5부위 정답 좌표)
part_id TEXT PRIMARY KEY
cat_id TEXT REFERENCES cats
part_type TEXT                   -- "tail" | "ears" | "paw" | "eyes" | "back"
x_ratio NUMERIC                  -- 0.0 ~ 1.0 (풍경 가로 비율)
y_ratio NUMERIC                  -- 0.0 ~ 1.0 (풍경 세로 비율)
hit_radius NUMERIC               -- 탭 인정 반경 (0.04 ~ 0.08)

-- hits (사용자 발견 기록)
user_id TEXT
part_id TEXT REFERENCES cat_parts
found_at TIMESTAMPTZ
seconds_to_find INTEGER          -- 풍경 진입 ~ 발견까지 (난이도 측정용)

-- home_slots (보금자리 가구 영구 저장)
user_id TEXT
slot_index SMALLINT              -- 0 ~ 4
furniture_id TEXT
chosen_at TIMESTAMPTZ
```

## G. 운영 워크플로 (Day 5 콘텐츠 파이프라인) — v1.1 갱신

> ⚠️ 부위별 PNG 따로 생성 → niji 7 통제 어려움 (안경 셰이프, 풀바디 자동 등장 등). 폐기.
> **새 전략: 풀바디 1자산 + Figma 사물 마스킹으로 모든 부위 추출.**

### 콘텐츠 1세트 = 4가지 자산만

1. **빈 풍경 PNG** (`--no cats` 적용)
2. **고양이 풀바디 PNG** 1~2장 (누운 자세 메인, 앉기 v2)
3. **가구 3종 PNG** (3중 택1용)
4. **풍경 마스킹 사물 정의** (Figma 레이어)

### 워크플로 (1풍경)

```
1. 빈 풍경 미드저니 생성
2. Figma 캔버스에 풍경 배치 (9:16)
3. 풀바디 고양이 PNG 복사 5장 → 각 5분할 영역에 배치
4. 각 복사본을 풍경 사물(통·지붕·벤치·아치 등)로 마스킹
   → 부위 1개만 보이게 (꼬리/귀/발/눈/등)
5. 보이는 부위 중심 좌표 → JSON 추출 (Figma plugin)
6. Supabase scenes/cats/cat_parts insert
7. 풀바디 마스킹 없는 원본 1장 → 보상 등장용
8. 가구 3종 metadata 등록
```

→ **1풍경 운영 ≤ 5분 목표** (Sprint1 Day 5 기준)
→ 풀바디 자산 1개 = 부위 5종 + 보상 등장 + 보금자리 스프라이트 = **8용도**

### 자산 재활용

- 같은 풀바디 1마리 = 여러 풍경 재활용 가능 (1마리 = 1주 시리즈)
- 새 고양이 = 풀바디 1장 새로 생성 (색/무늬만 변주)
- 풍경별 마스킹 위치만 바뀜 → 콘텐츠 변주 비용 최소

## I. 게임 메커니즘 확정 (v1.3, 2026-05-18 오후)

> **Trinity 결정**: "동일한 고양이가 각 부위별로 여러 군데 숨겨져 있으면 좋겠음. 1개만 찾는 건 난이도가 너무 쉬움."

### 확정 메커니즘 (옵션 C)

- 오늘의 고양이 = **1마리** (Sprint1.md 정통)
- 그 1마리가 **풍경의 사물에 의해 자연스럽게 가려진 상태**로 통합 배치
- 5부위만 풍경 곳곳에서 노출 (꼬리·귀·발·얼굴·등)
- 사용자가 5부위 위치 탭 → 5개 다 찾으면 → 마스킹 해제 → 풀바디 등장
- 매일 1마리 도감 영구 거주, 100마리 컬렉션 = ~100일

### 자산 구조

- 풍경 + 1마리 통합 PNG **1장** (niji 7만으로는 통제 어려움 확인됨)
- 풀바디 보상 = 통합 풍경에서 고양이 부분 크롭 + 확대 또는 별도 생성
- 가구 3종 PNG

### 도구 선택 (도구 검증 중)

- 1마리 강제 + 자연 가림 = **Nano Banana / GPT-Image-2** 등 reference·정확성 모델 필요
- niji 7은 풍경 톤 생성에만 활용 (1마리 통제 X)

## K. 시즌 메커니즘 (v1.5, 2026-05-20)

> **Trinity 결정 (2026-05-20)**: "고양이 한 마리당 가구 컬렉션을 완성해서 인테리어를 완성하는 것을 하나의 시즌으로." 가구·고양이 무제한 누적 → 시즌 단위로 묶음.

### K.1 시즌 정의

```
1 시즌 = 1 고양이 = 7 씬 = 7 가구 카테고리 = 1 인테리어 완성
```

- 시즌 1마리 고양이가 7일 동안 7개 다른 씬에 등장
- 매 씬 5부위 발견 → 해당 씬의 가구 카테고리에서 3 variant 중 1개 선택
- 7씬 클리어 = 7개 카테고리 가구 모두 모음 = **그 고양이의 보금자리 인테리어 완성**
- 시즌 완료 = 도감(보금자리 갤러리)에 영구 적립 + 다음 시즌 해금
- 100일 = 약 14 시즌 = 14 고양이 = 14 인테리어

### K.6 시즌 간 가구 풀 정책 (v1.6.2, 2026-05-20)

**룰:**

- **베이스 풀**: 시즌 1 단풍이/치즈에서 만든 21 variant (7 카테고리 × 3) = 영구 베이스
- **시즌마다 신규 1~2 variant** / 7 카테고리 중 2~3 카테고리에 추가
- **3중 택1 구성**: 베이스에서 1~2개 + 그 시즌 신규에서 1~2개 혼합
- 시즌 풀 = `베이스 + 그 시즌 신규` (이전 시즌 신규는 다음 시즌부터 베이스로 편입)

**효과:**

- 사용자: 매 시즌 "이번 시즌만의 신상 1~2개" 등장 → retention 강화
- 운영: 시즌당 미드저니 7~14회 생성 = 감당 가능
